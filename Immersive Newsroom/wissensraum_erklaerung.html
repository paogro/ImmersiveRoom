01 · Überblick

Systemarchitektur

Wissensraum ist eine visionOS-App für die Apple Vision Pro. Nutzer:innen betreten einen immersiven 3D-Raum und navigieren durch Wissensthemen (Sport, Natur, Technik, Politik), die als schwebende Panels im Raum erscheinen. Hinter jedem Leaf-Thema steht ein aktueller Nachrichtenartikel, der live aus einer Cloud-Datenbank geladen wird.

02 · Backend

Die Datenbank – Supabase & PostgreSQL

Das Backend läuft auf Supabase, einer Open-Source-Plattform, die eine PostgreSQL-Datenbank mit einer automatisch generierten REST-API kombiniert. Es gibt keine eigene Server-Logik – die App spricht direkt über das Supabase-SDK mit der Datenbank.

Datenbankstruktur

Tabelle / View Felder (Auswahl) Zweck
topics id, name, parent_id, level Hierarchie aller Themen (Baum-Struktur). Level 1 = Hauptkategorien (Sport, Natur …), tiefere Levels = Unterthemen.
published_news_view topic_id, headline, description, source_url, published_at Datenbankview mit allen freigegebenen Nachrichtenartikeln, verknüpft über topic_id mit dem jeweiligen Leaf-Thema.

Die Themen bilden einen Baum: Jedes Thema kennt seinen Elternknoten über parent_id. Hauptkategorien haben parent_id = NULL und level = 1. Unterthemen verweisen auf ihren Elternknoten. So kann die Hierarchie beliebig tief gestaffelt werden, ohne die App anfassen zu müssen.

💡
Wichtig: Die Artikel-Inhalte stehen nicht in der topics-Tabelle, sondern in einer separaten published_news_view. Das erlaubt es, pro Thema mehrere Artikel zu pflegen und unabhängig zu veröffentlichen – die App zeigt immer den aktuellsten.

Das Swift-Datenmodell

struct Thema: Codable, Identifiable { let id: UUID let name: String let parentId: UUID? // nil → Hauptkategorie let level: Int // 1 = Top-Level, 2+ = Unterthemen enum CodingKeys: String, CodingKey { case parentId = "parent_id" // snake_case → camelCase case level, name, id } }

Das CodingKeys-Enum mappt automatisch die snake_case-Spaltennamen der Datenbank auf Swift-konforme camelCase-Properties. Das JSON wird vom SDK typensicher dekodiert – kein manuelles Parsen nötig.

03 · Frische Inhalte

Warum die Inhalte immer aktuell sind

Die App speichert keine Themen oder Artikel lokal. Es gibt weder eine lokale Datenbank noch einen Offline-Cache. Stattdessen wird bei jedem App-Start und bei jeder Navigationsentscheidung live aus der Cloud geladen.

✅
Prinzip: Zero-Cache-Architektur. Jedes Mal, wenn ein Raum betreten oder ein Thema aufgeklappt wird, geht eine frische HTTP-Anfrage an Supabase. Was in der Datenbank steht, erscheint sofort in der Brille – ohne App-Update.

Das gilt für zwei Arten von Inhalten:

🌳

Themenstruktur

Beim Betreten eines Raums lädt getHauptkategorien() alle Level-1-Themen. Beim Aufklappen eines Panels folgt sofort getUnterthemen(vonThemaId:) für die nächste Ebene. Neue Kategorien erscheinen ohne Code-Änderung.

📰

Nachrichtenartikel

Beim Öffnen des Lesemodus lädt getNeuesteNews(fuerTopicId:) den neuesten freigegebenen Artikel aus published_news_view. Redakteure können Artikel jederzeit aktualisieren – der Nutzer sieht sie beim nächsten Öffnen.

// Neuesten Artikel laden – immer frisch aus der DB func getNeuesteNews(fuerTopicId id: UUID) async throws -> NewsArtikel? { let response: [NewsArtikel] = try await client .from("published_news_view") .select("headline, description, source_url, published_at …") .eq("topic_id", value: id.uuidString) .order("published_at", ascending: false) // aktuellster zuerst .limit(1) .execute().value return response.first }
⚡
Lazy Loading: Unterthemen werden erst geladen, wenn der Nutzer ein Panel antippt – nicht vorher. Das spart Bandbreite und hält die initiale Ladezeit kurz. Das Panel erscheint sofort mit einem Ladeindikator, der Artikel füllt sich nach.

Schritt für Schritt

1
Nutzer wählt Kategorie im Hauptmenü
ContentView ruft ThemenService.getHauptkategorien() auf und zeigt Sport, Natur, Technik, Politik als Buttons. Tipp setzt AppModel.ausgewaehltesThema.
2
ImmersiveSpace öffnet sich
ImmersiveView liest appModel.ausgewaehltesThema.name und wählt den passenden GenericRoomView(skyboxTextureName:) aus.
3
Erste Ebene wird geladen
GenericRoomView.ladeErsteEbene() ruft getUnterthemen(vonThemaId:) für die gewählte Hauptkategorie auf. Die Unterthemen landen in aktuelleThemen.
4
RealityKit rendert die Panels
Jedes Thema aus aktuelleThemen erzeugt ein SwiftUI-Attachment in der RealityView. Das update-Closure platziert es im 3D-Raum und setzt InputTargetComponent für Tap-Erkennung.
5
Tipp → Unterthemen → Artikel
Tipp auf ein Panel lädt die nächste Ebene. Tipp auf ein Leaf-Thema (ohne Kinder) öffnet den Lesemodus: getNeuesteNews(fuerTopicId:) holt den Artikel, LesePanelView zeigt ihn an.

Das AppModel als zentrale Schaltstelle

AppModel ist die einzige Datenquelle für den gesamten App-Zustand. Es ist als @Observable-Klasse implementiert und wird über SwiftUIs .environment()-Modifier in alle Views injiziert. Ändert sich ausgewaehltesThema, reagiert ImmersiveView sofort – kein manuelles Reload, kein Callback nötig.

@MainActor @Observable class AppModel { var ausgewaehltesThema: Thema? = nil // gesetzt → Raum wechselt automatisch var isImmersiveOpen: Bool = false var immersiveSpaceState: ImmersiveSpaceState = .closed }
05 · 3D-Welt

Die 3D-Welt: RealityKit & Skybox

Die gesamte 3D-Umgebung wird in GenericRoomView mit RealityKit und SwiftUI Attachments aufgebaut. Dieser eine View-Typ wird für alle vier Räume verwendet – nur das Skybox-Bild und der Ambient-Sound unterscheiden sich.

🌐

Skybox-Kugel

Eine Sphere mit Radius 50m, von innen texturiert (X-Achse gespiegelt, sodass das Bild nach innen zeigt). Equirektangulare 360°-Fotos bestimmen die Atmosphäre jedes Raums. Die Kugel folgt der Kopfposition, sodass der Nutzer nie an den Rand stößt.

🎴

SwiftUI Attachments

Die Themen-Panels sind keine reinen RealityKit-Objekte, sondern SwiftUI-Views, die über Attachment(id:) in die 3D-Szene eingebettet werden. Das erlaubt normale SwiftUI-Elemente (Text, Button, ScrollView) direkt im 3D-Raum.

💫

Ring-Karussell

Die Panels sitzen auf einem unsichtbaren 3D-Ring um den Nutzer herum. Dreht man den Ring, rotieren alle Karten gleichzeitig wie ein Zahnrad. Wenige Karten fächern sich frontal auf; viele legen sich als Vollkreis um den Raum.

🔊

Ambient Sound

Jeder Raum hat einen optionalen Loop-Sound (z.B. Stadion-Atmo für Sport), der über AmbientAudioComponent raumfüllend abgespielt wird – nicht ortsgebunden, sondern den Nutzer ringsum umhüllend.

Navigationsmodi

Modus Was der Nutzer sieht Technisch
Galerie-Modus Unterthemen der gewählten Kategorie als drehbares Ring-Karussell. Die Frontkarte ist leicht vergrößert (1.18×). fokusThema == nil; Panels aus aktuelleThemen; Ring-Rotation über ringEntity.transform.
Fokus-Modus Kinder des gewählten Themas erscheinen. Breadcrumb-Leiste unten zeigt den Navigationspfad. fokusThema != nil; Panels aus childrenThemen; Breadcrumbs als eigene Attachments.
Lesemodus Ein 700×500-Info-Panel mit Artikel-Überschrift, Zusammenfassung und Link zur Originalquelle. leseModusAktiv = true; Artikel aus published_news_view; Quellenlink öffnet QuelleWebView als separates Fenster.

Kopf-Tracking & Follow-Logik

Die App nutzt ARKit WorldTracking, um die aktuelle Kopfposition abzufragen. Ein 16ms-Loop zieht die gesamte UI sanft zur horizontalen Kopfposition nach (Lerp-Faktor 0.08), sodass die Panels immer in Sichtweite bleiben, ohne sprunghaft zu folgen.

06 · Interaktion

Gesten & Interaktionsdesign

Alle Interaktionen laufen über vier Gesten-Recognizer, die parallel auf der RealityView registriert sind. Die Brille erkennt Hände im Raum – es gibt keine physischen Buttons.

👆
Tippen (Tap)
SpatialTapGesture – navigiert tiefer, wählt Panels, schließt das Lese-Panel. Trifft immer das Entity, auf das der Gaze-Cursor zeigt.
↔
Wischen (Drag)
DragGesture(minDistance: 10) – dreht das Ring-Karussell. Nach Loslassen gleitet der Ring mit Momentum weiter und schnappt zur nächsten Karte.
✊
Halten (Hold)
DragGesture(minDistance: 0) + 450ms Timer – öffnet den Lesemodus. Das Panel leuchtet cyan und skaliert auf 1.06×, während der Timer läuft. Bewegung > 4cm bricht ab.
🤏
Kneifen (Pinch)
MagnifyGesture – skaliert die gesamte Szene (0.4×–2.5×). Nützlich, um einen besseren Überblick zu bekommen oder Details näher heranzuholen.

Hover-Feedback

Jedes Panel hat eine HoverEffectComponent mit blauem Gaze-Spotlight. Sobald der Nutzer ein Panel anschaut, leuchtet es auf – rein über das visionOS-System, ohne eigene Logik. Das gibt dem Nutzer immer visuelles Feedback darüber, welches Element gerade aktiv ist.

Hold-Geste im Detail

// Hold-Timer startet beim ersten Touch var holdGesture: some Gesture { DragGesture(minimumDistance: 0) .targetedToAnyEntity() .onChanged { value in if holdTask == nil { // Timer noch nicht gestartet holdTask = Task { try? await Task.sleep(for: .milliseconds(450)) await oeffneLesemodus() // Lesemodus öffnen holdTriggered = true // Tap unterdrücken } } // Bewegung > 4 cm → Timer abbrechen if bewegungZuGross(value) { holdTask?.cancel(); holdTask = nil } } }
🎯
Gesten-Konflikt gelöst: Hold- und Tap-Geste feuern am selben Panel. Das Flag holdTriggered = true sorgt dafür, dass der Tap nach einem Hold ignoriert wird – der Nutzer landet nicht versehentlich in der Navigation, wenn er eigentlich den Lesemodus wollte.

Breadcrumb-Navigation

Beim Navigieren tiefer in den Themenbaum erscheint eine Breadcrumb-Leiste unterhalb der Panels. Im eingeklappten Zustand stapeln sich die Elternthemen wie eine Kartendecke (jede dahinter etwas kleiner). Ein Tipp auf die Leiste klappt sie zu einem vertikalen Pfad aus – der Nutzer kann direkt auf jeden Vorfahren springen, ohne sich Schritt für Schritt zurückzuarbeiten.

Zusammenfassung

Das Gesamtbild

Wissensraum verbindet eine live-aktualisierte Cloud-Datenbank (Supabase) mit einer vollständig immersiven 3D-Umgebung (visionOS, RealityKit). Inhalte werden nie gecacht – jeder Raumbesuch lädt frische Themen und Artikel. Die Architektur ist so aufgebaut, dass Redakteure Themen und Artikel in der Datenbank pflegen können, ohne die App jemals anfassen zu müssen. Der Nutzer navigiert durch natürliche Gesten im Raum und kann jeden Artikel direkt im Headset lesen – inklusive Quellenlink im integrierten Browser.
